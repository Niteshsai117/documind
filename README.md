# DocuMind

A RAG-based Document Q&A system. Upload a PDF, and ask natural-language questions about it — answers are generated by a local **Ollama** model (llama3), grounded in the document's actual content via similarity search over chunk embeddings. No API keys, no paid services — everything runs on your machine.

## Architecture

```
                ┌───────────────┐
  PDF upload →  │  pdf_service  │  PyPDF2 text extraction + LangChain chunking
                └───────┬───────┘
                        ▼
                ┌────────────────────┐
                │ embedding_service  │  sentence-transformers (all-MiniLM-L6-v2)
                └───────┬────────────┘
                        ▼
                ┌────────────────────┐
                │ retrieval_service  │  ChromaDB persistent vector store
                └───────┬────────────┘
                        ▼
   Question  →  ┌────────────────────┐
                │     qa_service     │  query normalization + similarity search +
                │                    │  rerank-and-trim + Ollama (llama3, local)
                └───────┬────────────┘
                        ▼
                     Answer + sources
```

Each layer is independent and only talks to the layer below it through a small interface (`services/*.py`), so e.g. swapping ChromaDB for another vector DB only touches `retrieval_service.py`.

## Tech Stack

- **FastAPI** + **uvicorn** — HTTP API and ASGI server
- **LangChain** (`langchain-text-splitters`) — recursive character-based chunking
- **PyPDF2** — PDF text extraction
- **sentence-transformers** — local embedding model (no API calls, runs on CPU)
- **ChromaDB** — persistent local vector store
- **Ollama** (`llama3`) — local, free answer generation — no API key, nothing leaves your machine

## Setup

### 1. Install Ollama and pull llama3

Install Ollama from [ollama.com/download](https://ollama.com/download), then pull the model:

```bash
ollama pull llama3
```

Make sure the Ollama server is running (it starts automatically on most installs; otherwise run `ollama serve` in a separate terminal).

### 2. Clone and create a virtual environment

```bash
python3.11 -m venv venv
source venv/bin/activate   # Windows: venv\Scripts\activate
```

### 3. Install dependencies

```bash
pip install -r requirements.txt
```

### 4. Configure environment variables (optional)

No API key is needed. If you want to override any defaults (e.g. use a different Ollama model):

```bash
cp .env.example .env
```

### 5. Run the server

```bash
uvicorn main:app --reload
```

The app starts on **http://localhost:8000**:

- Frontend: http://localhost:8000/
- Swagger UI (interactive API docs): http://localhost:8000/docs
- ReDoc: http://localhost:8000/redoc

First run will download the `all-MiniLM-L6-v2` embedding model (~80MB) — this happens once and is cached locally.

## API Reference

### `GET /health`

Liveness check.

```json
{ "status": "ok", "service": "documind" }
```

### `POST /upload`

Upload and index a PDF document. `multipart/form-data` with a `file` field.

```bash
curl -X POST http://localhost:8000/upload \
  -F "file=@/path/to/document.pdf"
```

**Response (200):**

```json
{
  "document_id": "3f9a1c2b...",
  "filename": "document.pdf",
  "chunk_count": 42,
  "message": "Document indexed successfully. Use the document_id with /ask."
}
```

**Errors:** `400` not a PDF · `413` exceeds size limit (default 25MB) · `422` unreadable/scanned/encrypted PDF.

### `POST /ask`

Ask a question about a previously uploaded document.

```bash
curl -X POST http://localhost:8000/ask \
  -H "Content-Type: application/json" \
  -d '{"document_id": "3f9a1c2b...", "question": "What is the main conclusion?"}'
```

**Response (200):**

```json
{
  "answer": "The main conclusion is ...",
  "sources": [
    { "page_number": 4, "score": 0.8123, "excerpt": "..." }
  ]
}
```

**Errors:** `404` unknown `document_id` · `503` Ollama unreachable or model not pulled (run `ollama pull llama3` / `ollama serve`).

## Configuration

All settings are environment variables (see `.env.example`):

| Variable | Default | Description |
|---|---|---|
| `OLLAMA_HOST` | `http://localhost:11434` | URL of the local Ollama server |
| `OLLAMA_MODEL` | `llama3` | Model used for answer generation (must be pulled via `ollama pull`) |
| `OLLAMA_MAX_TOKENS` | `1024` | Max output tokens per answer (`num_predict`) |
| `EMBEDDING_MODEL_NAME` | `all-MiniLM-L6-v2` | sentence-transformers model |
| `CHUNK_SIZE` / `CHUNK_OVERLAP` | `500` / `50` | Text splitter parameters (chars) |
| `CHROMA_PERSIST_DIR` | `./chroma_db` | Where ChromaDB stores its index |
| `RETRIEVAL_TOP_K` | `5` | Chunks retrieved per question, before reranking |
| `RERANK_TOP_N` | `3` | Chunks kept after reranking, sent to the LLM as context |
| `UPLOAD_DIR` | `./uploads` | Scratch directory for incoming PDFs |
| `MAX_UPLOAD_SIZE_MB` | `25` | Max PDF upload size |

## Project Structure

```
documind/
├── main.py                       # FastAPI app, routes, CORS, error handling
├── config.py                     # Settings (pydantic-settings, .env-driven)
├── requirements.txt
├── .env.example
├── services/
│   ├── pdf_service.py            # PDF parsing + chunking
│   ├── embedding_service.py      # Text embeddings (sentence-transformers)
│   ├── retrieval_service.py      # ChromaDB vector store operations
│   └── qa_service.py             # RAG pipeline: retrieval + Ollama (local LLM)
└── static/
    └── index.html                # Minimal upload/ask UI
```

## Notes

- Multiple documents can be uploaded; each gets its own `document_id`, and `/ask` queries are scoped to that document only.
- The vector store persists to disk (`CHROMA_PERSIST_DIR`), so indexed documents survive a server restart.
- Embeddings and answer generation both run fully locally — no API key, no external network calls, no cost.
